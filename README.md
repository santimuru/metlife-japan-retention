# Policyholder Retention Analytics — MetLife Japan (portfolio demo)

A working lapse-prediction + customer-segmentation pipeline framed for MetLife
Japan, with a single-page interactive dashboard.

## Open it

Double-click **`index.html`** (needs internet for the Inter font + ECharts CDN).

## What it shows

1. Japan life-insurance market context — **real, cited** figures (LIAJ Fact Book 2024, JILI 2024, OECD, MIC).
2. A gradient-boosting **lapse-risk model** vs a logistic challenger, compared by **5-fold CV** (they tie — reported honestly), with permutation importance **and SHAP** (global + one local explanation).
3. **Model performance** — ROC, calibration curve, gains/lift, cross-sectional lapse rate by duration, risk-decile table.
4. **K-means customer segmentation** (value vs. risk), silhouette-checked, with a retention play per segment.
5. A **retention business case** with stated assumptions + a **sensitivity grid** and break-even (targeting, uplift, cost → ROI).
6. A **fairness audit** of the targeting flag (AUC + selection rate by age and income band).
7. **Proposed production architecture** (Azure ML, MLOps) — explicitly labelled not-yet-built, vs SHAP/fairness which are built here.

## Data honesty

Market context is real and sourced. The policyholder-level book is **synthetic**
(60,000 rows) generated by a logistic process whose coefficients encode the
published _direction_ of each lapse driver (early-duration hazard, premium-to-income,
payment method/frequency, channel, riders, complaints, servicing gap, age).
Every model metric on the page is a genuine output of the fitted model — nothing
is hard-coded.

## Reproduce

```
python build_model.py     # generates data, trains models, writes dashboard_data.json
python build_html.py      # renders index.html (CSS + data inlined)
node run_check.js         # headless DOM/ECharts runtime check (no browser needed)
```

## Stack

Python · scikit-learn (HistGradientBoosting, LogisticRegression, KMeans, permutation
importance) · ECharts · BI Design System (light, minimal).
